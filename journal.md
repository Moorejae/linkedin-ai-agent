Today's Engineering Session:

We encountered a major problem with Gemini's free tier quotas causing crashes when processing complex prompts. To solve this, we built a robust API Key Rotation Engine. We pooled 14 API keys into the environment. Now, when we hit a 429 (Quota Exceeded) or a 400 (Invalid Key) error, the server intercepts it and seamlessly rotates to the next key without crashing.

We also encountered models randomly going offline or returning 503 errors (High Demand). To counter this, we designed a Dual-Axis Fallback Matrix. It automatically cascades from the most advanced model down to more stable ones depending on the error it receives, ensuring 100% uptime regardless of Google's server loads.

Finally, we tackled the "Amnesia" problem. Our AI Prompt Architect, "Eyeno", was learning great prompt structures but forgetting them because we host on Render with ephemeral disks. We implemented a "Source of Truth" persistence layer. We connected the Node server directly to the GitHub REST API. Now, whenever Eyeno learns a new prompt architecture, she commits and pushes her Markdown memory file straight to our GitHub repository. Her brain literally grows on GitHub in real-time.
